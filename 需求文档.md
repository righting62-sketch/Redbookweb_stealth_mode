# 小红书网页版样式美化扩展 — 需求文档

## 1. 需求背景

小红书网页版（www.xiaohongshu.com）页面中存在大量红色品牌元素，包括导航栏Logo、浏览器标签页Favicon、"关注"按钮等。这些红色元素色彩鲜艳、辨识度极高，在公共场合或办公环境下浏览时非常显眼，容易引起他人注意，造成隐私尴尬。

因此需要开发一款浏览器扩展，在用户打开小红书网页时，自动将页面上的红色品牌元素替换为透明色/白色，使页面视觉上更加低调，降低被他人发现的风险。

## 2. 需求目标

- **一打开就生效**：用户访问小红书网页时，无需任何手动操作，红色元素自动变色
- **透明化/白色化**：将所有红色品牌元素替换为透明色或白色，与页面背景融为一体
- **不影响功能**：仅修改视觉样式，不改变页面功能和交互逻辑
- **轻量无感**：扩展运行时不消耗额外性能，用户无感知

## 3. 目标页面

- 域名：`www.xiaohongshu.com` 及其子域名
- 覆盖所有小红书网页端页面（首页、搜索页、用户主页、笔记详情页等）

## 4. 需要处理的红色元素

### 4.1 顶部导航栏 Logo

| 属性 | 说明 |
|------|------|
| 位置 | 页面左上角导航栏 |
| 当前样式 | 红色"小红书"图标 + 红色文字 |
| 目标样式 | 透明/白色，与导航栏背景融为一体 |

### 4.2 浏览器标签页 Favicon

| 属性 | 说明 |
|------|------|
| 位置 | 浏览器标签页标题左侧 |
| 当前样式 | 红色方块图标 |
| 目标样式 | 透明或白色图标，不显眼 |

### 4.3 "关注"按钮

| 属性 | 说明 |
|------|------|
| 位置 | 用户主页头像下方、笔记卡片中 |
| 当前样式 | 红色背景 + 白色文字 |
| 目标样式 | 白色/灰色背景 + 灰色文字，与普通按钮样式统一 |

### 4.4 其他红色元素

| 元素 | 说明 |
|------|------|
| 通知红点 | 导航栏消息/通知图标上的红色未读标记 |
| 点赞图标 | 笔记卡片中已点赞状态下的红色爱心 |
| 品牌色装饰 | 页面中所有使用小红书品牌色（#FE2C55 / #FF2442 等）的装饰性元素 |
| Tab 选中态 | 底部导航或分类Tab选中时的红色下划线/高亮 |

## 5. 技术方案

### 5.1 方案概述

采用 **Chrome 浏览器扩展（Manifest V3）** 实现，通过 Content Script 在页面加载时自动注入自定义 CSS 样式，覆盖小红书页面中的红色元素。

### 5.2 项目结构

```
redbookweb_change/
├── manifest.json          # 扩展配置文件（Manifest V3）
├── content.js             # 内容脚本：注入CSS + 替换Favicon
├── styles.css             # 覆盖样式表
└── icons/                 # 扩展图标目录
    ├── icon16.png
    ├── icon48.png
    └── icon128.png
```

### 5.3 实现原理

#### 5.3.1 CSS 样式覆盖（styles.css）

通过高优先级 CSS 规则覆盖红色元素：

- **Logo 处理**：对 Logo 容器设置 `opacity: 0` 使其完全透明，或使用 `filter: brightness(0) invert(1)` 将其变为白色
- **按钮处理**：重写"关注"按钮的 `background-color`、`border-color`、`color` 等属性，将红色替换为灰色系
- **通用红色处理**：对所有红色系背景色（`#FE2C55`、`#FF2442`、`#FF2A4B` 等）使用 `!important` 强制覆盖
- **红点处理**：覆盖通知红点的背景色为灰色

#### 5.3.2 Favicon 替换（content.js）

通过 JavaScript 动态操作 DOM：

1. 查找页面中的 `<link rel="icon">` 或 `<link rel="shortcut icon">` 标签
2. 将其 `href` 替换为一个透明/白色的 base64 图标
3. 使用 `MutationObserver` 监听 DOM 变化，防止小红书动态修改回原图标

#### 5.3.3 注入时机

- `manifest.json` 中配置 `"run_at": "document_start"`，确保 CSS 在页面渲染前注入
- 避免页面闪现红色元素（FOUC — Flash of Unstyled Content）

### 5.4 关键技术点

| 技术点 | 方案 |
|--------|------|
| 扩展框架 | Chrome Manifest V3 |
| CSS 注入 | Content Script 动态注入 `<style>` 标签 |
| 优先级保证 | 所有覆盖规则使用 `!important` |
| Favicon 持久化 | `MutationObserver` 监听 `<link>` 标签变化 |
| 域名匹配 | `matches: ["*://www.xiaohongshu.com/*"]` |
| 页面加载时机 | `run_at: "document_start"` |

### 5.5 小红书品牌色参考

| 颜色名称 | 色值 | 用途 |
|----------|------|------|
| 小红书红（主色） | `#FE2C55` | Logo、关注按钮、品牌标识 |
| 深红 | `#FF2442` | 按钮 hover 态、重要标识 |
| 浅红/粉红 | `#FF2A4B` | 标签、装饰元素 |

## 6. 使用方式

1. 打开 Chrome 浏览器，进入 `chrome://extensions/`
2. 开启右上角"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择 `redbookweb_change` 目录
5. 扩展加载后，访问 `www.xiaohongshu.com` 即可看到效果

## 7. 验收标准

- [ ] 打开小红书网页，导航栏 Logo 变为透明/白色
- [ ] 浏览器标签页 Favicon 变为透明/白色
- [ ] 页面中"关注"按钮背景变为灰色/白色
- [ ] 通知红点变为灰色
- [ ] 页面无红色元素闪现（FOUC）
- [ ] 页面所有功能正常可用（点击、滚动、搜索等）
- [ ] 刷新页面后效果依然生效

## 8. 风险与注意事项

- **小红书前端更新**：小红书可能会修改页面结构和CSS类名，导致选择器失效。需要定期检查和更新选择器
- **动态加载内容**：小红书使用React/Vue等框架，部分内容是动态渲染的，CSS选择器需要覆盖动态生成的元素
- **DOM 监听**：Favicon 可能被小红书动态修改，需要持续监听
- **性能影响**：MutationObserver 需合理配置，避免影响页面性能
